\subsection{Detailed Description}
Port base class. This class is a base class of RTC::Port. RTC::Port inherits a concept of RT-\/Component, and can be regarded as almost the same as it. In the concept of RT-\/Component, Port is attached to the component, can mediate interaction between other components and usually is associated with some interfaces. Component can provide or require interface for outside via Port, and the Port plays a role to manage the connection. 

Concrete class of Port assumes to be usually created at the same time that RT-\/Component's instance is created, be registerd to RT-\/Component after provided and required interfaces are registerd, and function as accessible Port from outside. 

RTC::Port provides the following operations as CORBA interface:

[CORBA interface] Connect the Port 

This operation establishes connection according to the given ConnectionProfile inforamtion. This function is premised on calling from mainly application program or tools.

\begin{DoxyPrecond}{Precondition}
To establish the connection among Ports of RT-\/Components, application programs must call this operation giving ConnectorProfile with valid values as an argument.

[CORBA interface] Notify the Ports connection 

This operation is usually called from other ports' \doxyref{connect()}{p.}{classRTC_1_1PortBase_a139d07d2e94f7e793aedf4aa24b92462} or \doxyref{notify\_\-connect()}{p.}{classRTC_1_1PortBase_a9bcc3c981a6649f37f3ad7eac31e5766} operations when connection between ports is established. This function is not premised on calling from other functions or application programs.

According to the template method pattern, the \doxyref{notify\_\-connect()}{p.}{classRTC_1_1PortBase_a9bcc3c981a6649f37f3ad7eac31e5766} calls \char`\"{}publishInterfaces()\char`\"{} and \char`\"{}subsctiveInterfaces()\char`\"{} functions, which are premised on implementing in the subclasses. The processing sequence is as follows.


\begin{DoxyItemize}
\item \doxyref{publishInterfaces()}{p.}{classRTC_1_1PortBase_acf31878c5912f56c122aaa2310e182de}: Publishing interface information
\item \doxyref{connectNext()}{p.}{classRTC_1_1PortBase_acd7017c12c99c59c191d1269b40a7423}: Calling \doxyref{notify\_\-connect()}{p.}{classRTC_1_1PortBase_a9bcc3c981a6649f37f3ad7eac31e5766} of the next port
\item \doxyref{subscribeInterfaces()}{p.}{classRTC_1_1PortBase_afce755069836c1ee637784e2a9e5a02b}: Subscribing interface information
\item Storing connection profile
\end{DoxyItemize}

According to the order of port's references stored in the ConnectorProfile::ports, publishing interface information to all the ports and subscription interface information from all the ports is performed by \char`\"{}notify\_\-connect()\char`\"{}s. This cascaded call never aborts in the halfway operations, and calling sequence shall be completed for all the ports.

\begin{DoxyPrecond}{Precondition}
\doxyref{notify\_\-connect()}{p.}{classRTC_1_1PortBase_a9bcc3c981a6649f37f3ad7eac31e5766} calls \doxyref{notify\_\-connect()}{p.}{classRTC_1_1PortBase_a9bcc3c981a6649f37f3ad7eac31e5766} for the port's reference that is stored in next of this port's reference in the sequence of the ConnectorProfile::ports. Therefore the reference of this port shall be stored in the ConnectorProfile::ports. If this port's reference is not stored in the sequence, BAD\_\-PARAMETER error will be returned, except the return code is overwritten by other operations.

UUID shall be set to ConnectorProfile::connector\_\-id as a unique identifier when this operation is called. Usually, connector\_\-id is given by a \doxyref{connect()}{p.}{classRTC_1_1PortBase_a139d07d2e94f7e793aedf4aa24b92462} function and, the behavior is undefined in the case of a null character.
\end{DoxyPrecond}
\begin{DoxyPostcond}{Postcondition}
ConnectorProfile::name, ConnectorProfile::connector\_\-id, ConnectorProfile::ports are invariant, and they are never rewritten by \doxyref{notify\_\-connect()}{p.}{classRTC_1_1PortBase_a9bcc3c981a6649f37f3ad7eac31e5766} operations.

In order to transfer interface information to other ports, interface property information is stored into the ConnectorProfile::properties.

[CORBA interface] Notify the Ports disconnection 

This operation is invoked between Ports internally when the connection is destroied. Generally it is not premised on calling from application programs or \doxyref{RTC}{p.}{namespaceRTC} objects except Port object.

According to the template method pattern, the \doxyref{notify\_\-disconnect()}{p.}{classRTC_1_1PortBase_ae78bb7882d22bca794bd0faa78452aca} calls unsubsctiveInterfaces() function, which are premised on implementing in the subclasses. The processing sequence is as follows.


\begin{DoxyItemize}
\item Searching ConnectorProfile
\item Calling \doxyref{notify\_\-disconnect()}{p.}{classRTC_1_1PortBase_ae78bb7882d22bca794bd0faa78452aca} for the next port
\item Unsubscribing interfaces
\item Deleting ConnectorProfile
\end{DoxyItemize}

\doxyref{notify\_\-disconnect()}{p.}{classRTC_1_1PortBase_ae78bb7882d22bca794bd0faa78452aca} notifies disconnection to all the ports by cascaded call to the stored ports in the ConnectorProfile::ports in order.

\begin{DoxyPrecond}{Precondition}
The port shall store the ConnectorProfile having same id with connector\_\-id.
\end{DoxyPrecond}
\begin{DoxyPostcond}{Postcondition}
If ConnectorProfile of same ID with connector\_\-id does not exist, it returns BAD\_\-PARAMETER error.

For the cascaded call, this operation calls noify\_\-disconnect() for the port that is stored in the next of this port in the ConnectorProfile::ports. If the operation call raises exception for some failure, it tries to call \doxyref{notify\_\-disconnect()}{p.}{classRTC_1_1PortBase_ae78bb7882d22bca794bd0faa78452aca} and skips until the operation succeeded. If none of operation call succeeded, it returns RTC\_\-ERROR.

At the end of \doxyref{notify\_\-disconnect()}{p.}{classRTC_1_1PortBase_ae78bb7882d22bca794bd0faa78452aca} operation for the first port stored in the ConnectorProfile::ports sequence, the related ports' \doxyref{notify\_\-disconnect()}{p.}{classRTC_1_1PortBase_ae78bb7882d22bca794bd0faa78452aca} invocations complete.
\end{DoxyPostcond}
