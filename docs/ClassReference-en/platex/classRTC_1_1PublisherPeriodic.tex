\subsection{Detailed Description}
\doxyref{PublisherPeriodic}{p.}{classRTC_1_1PublisherPeriodic} class. Publisher for periodic data transmitting. Usually this class object exists in a Connector object, and it is associated with a buffer and a consumer. This publisher periodically gets data from the buffer and publish it into the consumer.

Initialization. 

This function have to be called before using this class object. Properties object that includes certain configuration information should be given as an argument. At least, a numerical value of unit of Hz with the key of \char`\"{}dataport.publisher.push\_\-rate\char`\"{} has to be set to the Properties object of argument. The value is the invocation cycle of data sending process. In case of 5 ms period or 200 Hz, the value should be set as 200.0. False will be returned, if there is no value with the key of \char`\"{}dataport.publisher.push\_\-rate\char`\"{}.

The following options are available.


\begin{DoxyItemize}
\item publisher.thread\_\-type: Thread type (string, default: default)
\item publisher.push\_\-rate: Publisher sending period (numberical)
\item publisher.push\_\-policy: Push policy (all, fifo, skip, new)
\item publisher.skip\_\-count: The number of skip count in the \char`\"{}skip\char`\"{} policy
\item measurement.exec\_\-time: Task execution time measurement (enable/disable)
\item measurement.exec\_\-count: Task execution time measurement count (numerical, number of times)
\item measurement.period\_\-time: Task period time measurement (enable/disable)
\item measurement.period\_\-count: Task period time measurement count (number, count)
\end{DoxyItemize}

Write data. 

This function writes data into the buffer associated with this Publisher. If a Publisher object calls this function, without initializing correctly such as a consumer, a buffer, listeners, etc., error code PRECONDITION\_\-NOT\_\-MET will be returned and no operation of the writing to a buffer etc. will be performed.

Since writing into the buffer and sending data to \doxyref{InPort}{p.}{classRTC_1_1InPort} are performed asynchronously, occasionally this function returns return-\/codes such as CONNECTION\_\-LOST and BUFFER\_\-FULL that indicate the result of sending data to \doxyref{InPort}{p.}{classRTC_1_1InPort}. In this case, writing data into buffer will not be performed.

When publisher writes data to the buffer, if the buffer is filled, returns error, is returned with timeout and returns precondition error, error codes BUFFER\_\-FULL, BUFFER\_\-ERROR, BUFFER\_\-TIMEOUT and PRECONDITION\_\-NOT\_\-MET will be returned respectively.

In other cases, PROT\_\-ERROR will be returned.
