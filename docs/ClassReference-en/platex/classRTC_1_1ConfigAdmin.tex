\subsection{Detailed Description}
\doxyref{ConfigAdmin}{p.}{classRTC_1_1ConfigAdmin} class. Class to manage various configuration information. Now terms for this class are defined as follows.


\begin{DoxyItemize}
\item Configurations: The configuration information for the RTCs.
\end{DoxyItemize}


\begin{DoxyItemize}
\item (Configuration) parameters: Configuration information that consists of a key-\/value pair. The \char`\"{}key\char`\"{} and the \char`\"{}value\char`\"{} are both stored as character string values in a \doxyref{coil::Properties}{p.}{classcoil_1_1Properties} variable in this class. The \char`\"{}key\char`\"{} is called the \char`\"{}configuration
   parameter name\char`\"{}, and the \char`\"{}value\char`\"{} is called the \char`\"{}configuration
   parameter value\char`\"{}.
\end{DoxyItemize}


\begin{DoxyItemize}
\item Configuration-\/sets: This is a list of configuration parameters, and it is distinguished by name (ID). The ID is called configuration-\/set ID.
\end{DoxyItemize}


\begin{DoxyItemize}
\item (Configuration) parameter variables: The variables to be referred when configuration parameters are actually used within the activity of an \doxyref{RTC}{p.}{namespaceRTC}. Each variable has each type.
\end{DoxyItemize}


\begin{DoxyItemize}
\item Active (configuration) set: This is the only configuration-\/set that is currently active. The parameter values of the active configuration-\/set are substituted into configuration variables in principle.
\end{DoxyItemize}

The following two configuration informations are stored in this class.


\begin{DoxyEnumerate}
\item A list of configuration-\/set
\item A list of configuration parameter variables
\end{DoxyEnumerate}

Basically, the purpose of this class is to set one of the configuration-\/set in the list of (1) into parameter variables of (2). Usually, configuration parameter variables manipulation is performed with two-\/phases of configuration-\/set setting and parameter variables setting.

The configuration-\/set manipulations are performed by the following functions.


\begin{DoxyItemize}
\item \doxyref{getConfigurationSets()}{p.}{classRTC_1_1ConfigAdmin_a9367d39f878ec8d817e8aabacd27deb6}
\item \doxyref{getConfigurationSet()}{p.}{classRTC_1_1ConfigAdmin_a51b2ad46ba5184fc1361dbd86ed2309a}
\item \doxyref{setConfigurationSetValues()}{p.}{classRTC_1_1ConfigAdmin_a31d4e545b48aa3ea2fc03109b301586d}
\item \doxyref{getActiveConfigurationSet()}{p.}{classRTC_1_1ConfigAdmin_a40c1de78caa9c8ff2ca02ee972f26bb9}
\item \doxyref{addConfigurationSet()}{p.}{classRTC_1_1ConfigAdmin_a35e38434ba8b409ae1207d81f4904a4d}
\item \doxyref{removeConfigurationSet()}{p.}{classRTC_1_1ConfigAdmin_aef99fcfb1f6ca7b09b85c10082236f71}
\item \doxyref{activateConfigurationSet()}{p.}{classRTC_1_1ConfigAdmin_a5215efd13ca14ed67cbbbd533556c0a3}
\end{DoxyItemize}

Modification, addition, deletion, acquisition and activation of configuration-\/set are performed by these functions. In order to reflect configuration-\/set, which is manipulated by these functions, on parameter variables that are used from \doxyref{RTC}{p.}{namespaceRTC} activities, the following \doxyref{update()}{p.}{classRTC_1_1ConfigAdmin_a858c419de43392804dd934dd62c1874a} functions are used .


\begin{DoxyItemize}
\item \doxyref{update(void)}{p.}{classRTC_1_1ConfigAdmin_a858c419de43392804dd934dd62c1874a}
\item \doxyref{update(const char$\ast$ config\_\-set)}{p.}{classRTC_1_1ConfigAdmin_a6a183d22dfa7c58ba5d21982aac34645}
\item \doxyref{update(const char$\ast$ config\_\-set, const char$\ast$ config\_\-param)}{p.}{classRTC_1_1ConfigAdmin_a4b1ce69c995d6a89d743cbfd38088de4}
\end{DoxyItemize}

Callback functors can be given to hook configuration operation. Operations to be hooked are as follows.


\begin{DoxyItemize}
\item ON\_\-UPDATE : when \doxyref{update()}{p.}{classRTC_1_1ConfigAdmin_a858c419de43392804dd934dd62c1874a} is called
\item ON\_\-UPDATE\_\-PARAM : when update(param) is called
\item ON\_\-SET\_\-CONFIGURATIONSET : when setConfigurationSet() is called
\item ON\_\-ADD\_\-CONFIGURATIONSET : when \doxyref{addConfigurationSet()}{p.}{classRTC_1_1ConfigAdmin_a35e38434ba8b409ae1207d81f4904a4d} is called

\item ON\_\-ACTIVATE\_\-CONFIGURATIONSET: when \doxyref{activateConfigurationSet()}{p.}{classRTC_1_1ConfigAdmin_a5215efd13ca14ed67cbbbd533556c0a3} is called
